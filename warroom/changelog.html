v3.4.30 ★ Major release 2026-06-19

Onboarding & Distribution

The biggest non-code release yet — a ground-up rework of how people discover, install, and run AgentRecall, for humans and agents alike. Plus a real security fix folded in from the community.

迄今最大的"非代码"版本 —— 从头重做了人与 agent 发现、安装、运行 AgentRecall 的整条路径，并合入了一个来自社区的真实安全修复。

📖

README, rewritten

Root README cut 579 → 198 lines — a scannable English landing page. The full reference is preserved verbatim as README.full.md.

Why: a landing page should convert in 30 seconds; the deep dive stays one click away.

中文：README 从 579 行精简到 198 行，做成可快速浏览的落地页；完整版原样保留为 README.full.md。

🌐

Bilingual docs

New README.zh-CN.md mirror with a one-click English · 中文 switch at the top of each file.

Why: first-class Chinese onboarding, not an afterthought.

中文：新增 README.zh-CN.md 镜像，顶部一键切换中英文。

🖥️

War Room in the repo — fully offline

The multi-page dashboard now lives at warroom/. ECharts, Cytoscape, 3 fonts and 8 brand icons are vendored — zero external requests.

Why: download → unzip → python3 -m http.server → open. No Node, no internet.

中文：仪表盘进仓库 warroom/，图表库/字体/图标全部本地化，完全离线可跑。

📦

Release pipeline

release.yml zips warroom/ on every v* tag and attaches ar-warroom-vX.Y.Z.zip to a GitHub Release (tag via env var, actions SHA-pinned).

Why: a versioned, downloadable dashboard for every release — the recommended path for Hermes / OpenClaw / OpenCode.

中文：打 v* tag 自动打包仪表盘为 zip 并挂到 Release，给每个版本一个可下载的离线面板。

🧹

Repo tidy

7 internal / QA folders (agent-prompts, eval, integrations, tests, wiki, workspace, scripts) consolidated under meta/.

Why: the GitHub root now shows only meaningful folders — .github · benchmark · commands · docs · packages · warroom · meta.

中文：7 个内部/QA 目录收进 meta/，仓库根目录只剩有意义的文件夹。

🔢

Version sync

3.4.27 → 3.4.30 everywhere: all 4 packages + internal deps + types.ts VERSION + SKILL.md + benchmark + codex-compat.

Why: one consistent version, no drift between manifest, code, and docs.

中文：版本号在所有包、依赖、源码常量、SKILL、benchmark 中统一为 3.4.30。

🔐

Security fix — community PR #18

Removed a hardcoded Supabase URL + anon key shipped as a default fallback in awareness.ts. It now gates on readSupabaseConfig() and returns nothing when unconfigured.

Why: unconfigured users were silently reading from a baked-in backend. Thanks to @leirt97.

中文：删除 awareness.ts 里写死的 Supabase 地址与 anon key —— 未配置的用户不再悄悄连到内置后端。

✅

Reviewed before shipping

A 5-agent round table reviewed the diff and caught 7 defects — including a shell-injection in the release workflow — all fixed before push.

Why: green CI ≠ correct. Independent review catches what self-verification misses.

中文：5 个 agent 圆桌评审，发现并修复 7 个问题（含 release 工作流的 shell 注入）后才推送。

Verified build · 0 errors consistency 10/10 funnel 18/18 heeded-guard 5/5 room-slug-guards 9/9 offline · 0 CDN refs
Governance: this release is staged on a pushed branch + PR. The irreversibles — npm publish ×4, the v3.4.30 tag, and the merge to main — are run by a human after review. Until the tag is pushed, the README's releases/latest download link points to an earlier release.
